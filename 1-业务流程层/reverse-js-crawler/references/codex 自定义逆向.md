  ## js-reverse MCP 使用要求

  ### 必须使用的场景
  1. 查看页面实际渲染内容和网络请求
  2. 追踪 JS 函数调用栈
  3. 获取运行时变量（Cookie / localStorage / 全局对象）
  4. Hook 函数或拦截请求（XMLHttpRequest / fetch）
  5. 模拟用户交互（点击/滚动/输入）
  6. 分析 WebSocket / SSE 实时通信
  7. 反混淆或格式化 JS 代码

  ### 核心操作模式
  ```javascript
  // 1. 打开页面并监控请求
  await page.goto(url);
  page.on('request', req => console.log(req.url(), req.headers()));

  // 2. 注入 Hook 拦截加密函数
  await page.evaluate(() => {
      window.originalEncrypt = window.encrypt;
      window.encrypt = function(...args) {
          console.log('encrypt called:', args);
          return window.originalEncrypt(...args);
      };
  });

  // 3. 获取运行时数据
  const cookies = await page.evaluate(() => document.cookie);
  const storage = await page.evaluate(() => JSON.stringify(localStorage));

  // 4. 执行加密函数并获取结果
  const sign = await page.evaluate((params) => {
      return window.generateSign(params);
  }, params);

  使用原则

  - 先动态后静态：先用 Hook/断点获取真实数据，再分析代码逻辑
  - 优先 Hook：直接拦截关键函数，而非盲目搜索代码
  - 记录完整：保存调用栈、输入参数、输出结果
