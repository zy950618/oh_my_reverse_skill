# 交付收尾清理与加密算法图谱规约

本规约解决两个收尾问题:

- 任务已经处理、验证、沉淀,但工作区还残留临时测试文件、历史评分、无用代码、过期注释和一次性调试产物。
- 加密/sign/token 逻辑只散落在聊天记录、脚本和断点里,没有形成一份整体算法细节图。

## 触发场景

完成以下任一动作后必须执行本规约:

- 已完成阶段 E 沉淀和阶段 F 一致性验证。
- 已定位、还原或复现 sign/token/fingerprint/risk/captcha token。
- 已改端点、字段、请求头、补环境、解混淆脚本、adapter、eval 或 fixtures。
- 准备向用户声明"完成"、"交付"、"收尾"。

## 清理原则

清理发生在验证之后,不是验证之前。不要删掉还没迁移的证据。

## 交付产物隔离原则

项目交付物、项目经验库和 SKILLS 能力层必须隔离:

- 项目交付物是用户可运行的单一交付文件或交付包,例如 adapter 服务包、接口脚本、runbook、schema。
- 项目经验库只保存真实接口测试确认后的精炼事实摘要、证据索引、失败模式、图谱、影响回归和监控要求。
- SKILLS 能力层只吸收成功经验库中被验证过的通用规则;交付项目代码不准直接进入 skill/evals/references 作为正向能力。
- blocked、adapter-only、negative baseline 只能作为 failure evidence、known-failure、test-log、human-review/refusal ledger 或负例 eval,不能参与正向 SKILLS 能力评分。
- 研发阶段材料、完整 HAR、完整 req/resp、token/cookie、订单参数、浏览器 profile 和临时脚本只保留在本地临时项目库或交付目录,不能进入 GitHub 上传面。

收尾时必须检查:

```text
Delivery / Memory / Skills Separation:
  project_delivery_artifact:
  experience_memory_path:
  delivery_status:
  skills_participation:
  mixed_code_in_memory: yes/no
  mixed_delivery_in_skills: yes/no
```

| 类型 | 处理 |
|---|---|
| 临时测试脚本 | 如果已被正式脚本/eval/fixture 替代,删除 |
| 临时输出 | 如 `.tmp`、一次性 HAR 转换中间产物、debug dump,迁移必要证据后删除 |
| 历史评分 JSON | 已被当前评分文档和 CI 产物替代的旧版本,删除或移出仓库 |
| 废代码 | 无引用、无测试、无交付用途,删除 |
| 废注释 | 过期 TODO、调试解释、与当前实现不符的注释,删除或改成事实注释 |
| 历史证据 | 不能直接删;先写入 `reverse-memory.md`、`knowledge-graph.md`、`impact-regression.md` 或 fixtures meta |
| 失败样本 | 不能直接删;先沉淀到 known-failures/test-log/eval/metrics |

## 禁止清理方式

- 不为"看起来干净"删除仍是唯一证据的 HAR、fixture、日志、脚本 hash、请求样本。
- 不删除用户未要求删除且与本次任务无关的改动。
- 不用大范围格式化掩盖真实 diff。
- 不删除能解释算法来源、字段来源、session 绑定或回归原因的证据。

## 必须产出的加密算法细节图

凡涉及加密/sign/token,最终必须有一份整体图,推荐路径:

```text
逆向工程经验库/domains/<domain>/encryption-algorithm-graph.md
```

如果没有真实 domain,先使用模板:

```text
逆向工程经验库/_templates/encryption-algorithm-graph.md
```

算法图至少包含:

- 入口:请求字段、header/body/query/cookie/storage key。
- 触发:业务函数、拦截器、SDK、事件或接口调用。
- 脚本:script URL、hash、etag、版本、source map 状态。
- 函数链:调用栈、模块 id、函数名或 offset。
- 输入:业务参数、时间、随机数、storage、cookie、设备/环境字段、会话态。
- 变换:排序、拼接、编码、hash、HMAC/RSA/AES/自定义混淆、压缩、base64/urlencode。
- 输出:字段名、格式、长度、TTL、绑定关系。
- 验证:浏览器样本、Node/Python 复现、replay 响应、JSON Pointer、失败码。
- 漂移:旧算法图与新算法图的差异、受影响端点、必跑回归。
