# 当前 SKILLS 评分与回测结果

评分日期：2026-04-23

## 评分口径说明

从本轮开始，本地评分不再使用“单一结构总分”，改为三段分：

- **结构分（40）**：标准目录、`SKILL.md`、frontmatter、`agents/openai.yaml`、`references/`、`evals/`、版本与治理信息。
- **实战分（40）**：触发边界、正负例、criteria 质量、行为闭环、相邻 Skill 路由边界、经验回写要求。
- **漂移分（20）**：回归/边界 eval、drift 策略、版本与 change log、按周期复测意识。

注意：当前“实战分”仍然主要是 **operational readiness（实战就绪度）**，不是线上真实成功率。真实命中率、真实完成率、周期漂移统计还需要后续单独沉淀。

## 本轮高分原因

### 共性高分原因

- 5 个核心 Skill 都具备标准结构：`SKILL.md`、`agents/openai.yaml`、`references/`、`evals/`、`governance.md`。
- 触发边界比较清楚，已覆盖正例、负例、回归/边界案例。
- `Workflow`、`Success Criteria`、`Boundaries`、`Governance` 基本完整。
- 已把版本、变更、drift、站点经验库写回纳入治理体系。

### 单项高分原因

- `website-314-api-delivery`：总控能力最完整，既有多阶段交付边界，也有 site memory / test-log mining / payment boundary。
- `reverse-js-crawler`：逆向链路清晰，负例和回归设计较成熟，交付闭环明确。
- `imperva-waf-reese84`：WAF 接受度与阶段性验证边界清楚，能避免“token 生成成功 = 业务成功”的误判。

- `skills-evaluation-governance`：治理规则最完整，已经具备准入、回测、漂移和评分标准骨架。

## 本轮失分 / 缺口

### 共性失分原因

当前失分并不在结构层，而在“真实进化证据层”：

- 缺少真实任务命中率统计。
- 缺少真实任务完成率统计。
- 缺少按周期记录的漂移指标与历史结果。
- 缺少单 Skill 的长期 metrics / reports / history 类产物。

### 单项缺口

- `website-314-api-delivery`：站点经验沉淀较强，但仍缺少按月/按版本的真实任务通过率与回退率。
- `reverse-js-crawler`：缺真实命中率、真实完成率、跨站点样本统计。
- `imperva-waf-reese84`：缺真实 challenge 变体的长期漂移记录和阶段性接受度趋势。
- `site-api-adapter`：站点记忆与失败模式证据弱于总控 Skill，市场路由类经验仍可继续扩充。
- `skills-evaluation-governance`：治理能力强，但“评分系统本身”的真实长期统计还没有独立样本仓。
