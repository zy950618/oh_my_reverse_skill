# AI 变更风险与回归校验规约

本规约只解决一个问题：AI 每次改动后，如何防止改 A 伤 B、无数据支撑、写死请求头/指纹、数据偏移、反扒误处理。

知识图谱节点和边的记录格式见 `14-知识图谱行程与关联规约.md`。

## Impact Record

每次改动都要在 `impact-regression.md` 追加记录：

```yaml
change_id:
date:
changed_node:
changed_edge:
change_type: add | modify | remove | deprecate
evidence:
direct_impact:
downstream_impact:
required_regression:
data_validation:
drift_risk:
rollback:
owner_notes:
```

## AI 风险与校验

| AI 风险 | 必须记录 | 必须回测 |
|---|---|---|
| 改 A 动 B | `direct_impact` + `downstream_impact` | 受影响 endpoint/eval |
| 发散思维 | 本次 `in_scope` 节点/边 | out-of-scope 不得新增实现 |
| 没数据支撑 | evidence + 事实等级 | 无证据只能写 assumed/unverified |
| 写死请求头 | request-field 来源、TTL、适用 scope | 同端点 replay + 失败路径 |
| 写死指纹 | protection 观察、稳定性、适用 scope | 不得把临时值写成固定规则 |
| 数据校验缺失 | response-field JSON Pointer | snapshot_diff / consistency_report |
| 数据偏移 | drift_risk + schema 变化类型 | schema_alert / affected fixtures |
| 反扒误处理 | protection 与 business-error 分开 | 403/429/503 与 200/error 分流验证 |

## 回测矩阵

| 改动类型 | 必跑 |
|---|---|
| endpoint URL / method / body | snapshot_replay + consistency_report + route-decisions |
| request headers / cookies / state | 单请求复现 + 失败路径 + session 隔离 |
| response mapper / schema | snapshot_diff + schema_alert + JSON Pointer 检查 |
| market / locale / currency | market-matrix + 同 stage endpoint replay |
| protection / fingerprint observation | 接受度验证 + 降速/隔离记录 |
| adapter route | smoke + negative + affected fixtures |
| eval / skill workflow | score_skills + ci_gate |

## 请求头和指纹规则

- 请求头不能只写常量，必须写来源：浏览器默认、服务端返回、JS 生成、storage 状态、人工配置。
- 请求头必须写 scope：domain、host、market、stage、auth_state、session。
- 指纹相关内容只记录观察和稳定性，不写绕过方案。
- 临时 cookie/token/session 值不能进入长期规则。
- 如果必须配置固定 header，必须说明为何稳定、如何验证、何时失效。

## 数据校验规则

- 每个交付字段必须能追到 JSON Pointer。
- 每个响应 mapper 改动必须跑 snapshot_diff。
- 每个 schema 变化必须标 compatible / breaking / unknown。
- 一致率低于阈值时，不允许通过扩大 ignore 字段刷分。
- 数据偏移必须进入 `known-failures.md` 或 `impact-regression.md`。

## 反扒处理规则

- 先区分业务错误、路由错误、参数错误、数据错误、保护触发。
- 不能把所有 403/空数据都归因于反扒。
- token/challenge endpoint 成功不等于 protected business API 接受。
- 反扒相关恢复只能写观察、降速、隔离、重录、失败记录；不写绕过方案。

## 禁止项

- 不允许没有 Impact Record 就声明完成。
- 不允许改实现后只跑 happy path。
- 不允许写死 header/cookie/fingerprint 临时值。
- 不允许把数据偏移解释成“正常波动”而没有 diff。
- 不允许把“反扒”当万能解释。
