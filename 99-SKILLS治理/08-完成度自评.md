# 完成度自评规约 (Completion Self-Score Spec)

每次声明"完成 / done / 交付"前,必须按本规约自评。这是 **Claude 自己用的 quality gate**,不是给用户看的。

Stop hook 检测到完成 marker 但没看到验证 marker 时会输出提醒(见 [post_task_reminder.py](../tools/post_task_reminder.py))。

## 为什么需要

历史教训:

| 版本 | 跳过的维度 | 后果 |
|---|---|---|
| v0.3.6 | 代码层 + 集成层 | 8 个新脚本里 6 个完全没跑过,`get_field()` 字段解析没单测,CI yaml 没本地校验 |
| v0.3.7 | 全部 5 维 | 用户两次追问后自读发现 5 个真实问题: 触发词漏、跨平台命令错、阶段命名不一致、README 标题断裂、规划模板标题陈旧 |
| v0.5.0 | 收尾层 | 验证通过后仍残留临时测试文件、历史输出、废代码、废注释,且 sign/token 逆向没有整体算法细节图 |

模式: **写完就说完成,几乎不做端到端验证**。本规约把验证强制化。

---

## 强制 6 维自评

每次声明完成前,6 维全打勾。打不上的列出来。

### 1. 代码层 (Code)

- [ ] 主路径端到端跑过一次 (不只是 lint / 静态检查)
- [ ] 改动的脚本独立跑过 (至少 dry-run)
- [ ] 新加的依赖在干净环境验证过 / 或文档明示"未在干净环境验证"

**反例**: v0.3.6 写了 cloak_recorder.py / snapshot_replay.py / snapshot_diff.py / consistency_report.py / recording_to_fixtures.py 一个都没跑过。

### 2. 文档层 (Docs)

- [ ] 自己写的所有文件至少完整读过一遍
- [ ] 代码块里的命令**至少抽查一条**实际跑通
- [ ] 表格 / 列表的对齐没断 / 标题层级没乱

**反例**: v0.3.7 写完 USAGE/INSTALL/TRIGGERS/README 共 ~600 行,声称完成时**一遍都没读**。

### 3. 集成层 (Integration)

- [ ] 内部链接 `./X.md` 目标存在
- [ ] 跨文件引用一致 (如 USAGE、README、06 规划模板对阶段数和阶段名的说法一致)
- [ ] 改完一个文件后,引用它的其他文件没失效
- [ ] 关键词 / 触发词在所属 SKILL.md 的 description 里真的存在

**反例**: v0.3.7 TRIGGERS 列了若干入口关键词,但对应 SKILL.md description 里没有这些词,用户用这些词不一定触发。

### 4. 回归层 (Regression)

- [ ] 改完核心脚本 (如 `score_skills.py`) 跑过所有依赖层 (1/2/4/5)
- [ ] 之前能跑通的命令现在还能跑
- [ ] CI yaml 改完跑过 `yamllint` 或对照过 schema
- [ ] 改端点/字段/请求头/状态/保护/实现/eval 后,已更新 knowledge-graph.md 和 impact-regression.md
- [ ] Impact Record 已列 direct_impact / downstream_impact / required_regression / data_validation

**反例**: 无大问题,但 v0.3.6 改 `score_skills.py` 时只跑了 1 层,其他依赖层是后来用户问才跑的。

### 5. 诚实层 (Honesty)

- [ ] 列出"做了什么"
- [ ] 列出"没验证什么"
- [ ] 关键结论按 observed / derived / assumed / unverified 分级
- [ ] 未把单接口、单 market、单 session 成功泛化成全链路成功
- [ ] 未声称支持未验证的并发、指纹、session/cache 复用
- [ ] 不把"代码写完"等同于"功能能用"
- [ ] 不把"评分高"等同于"产出对"

**反例**: v0.3.6 主动汇报"42.47"是好的(诚实化),但 v0.3.7 完全没说"零验证"。

### 6. 收尾层 (Cleanup)

- [ ] 已删除被正式脚本/eval/fixture 替代的临时测试文件。
- [ ] 已删除或移出仓库的旧评分、旧 debug dump、过期中间产物。
- [ ] 已清理无引用代码、过期 TODO、与当前事实不符的注释。
- [ ] 仍保留的 HAR/fixture/log/script hash 都有证据用途和保留理由。
- [ ] 涉及 sign/token/加密算法时,已写 `逆向工程经验库/domains/<domain>/encryption-algorithm-graph.md`。
- [ ] 已按 `17-交付收尾清理与加密算法图谱规约.md` 输出 cleanup ledger。

**反例**: 任务验证已通过,但仓库里还留着一次性调试脚本、过期评分 JSON、无引用 helper 和旧注释,下一轮 AI 又把旧数据当成当前证据。

## 完成度评分公式

| 6 维全勾 | 跳过 1 维 | 跳过 2 维 | 跳过 3 维 | 跳过 4-6 维 |
|---|---|---|---|---|
| 10/10 | 7/10 | 5/10 | 3/10 | ≤2/10 |

不允许"宣称完成 → 实际分 < 7"。低于 7 必须列具体跳过项 + 解释为什么不能现在做(如外部依赖未到位)。

## 验证 Marker (Stop hook 用)

Stop hook 在 transcript 里扫两组关键词。

**完成 marker** (任一命中 = 在说"我做完了"):
```
完成 / 做完 / 交付 / 提交 / 收尾 / 结束 / 完工 / 全部完成 / 已经做完 /
done / delivered / finished / wrapped up / shipped
```

**验证 marker** (任一命中 = Claude 真的验过):
```
验证 / verified / verify / 跑过 / 实测 / 自评 / 自检 / 自读 / 端到端 /
checklist / 6 维 / 六维 / spot check / smoke test / dry-run pass / cleanup ledger / 清理账本 / 加密算法图
```

**触发条件**: 命中 ≥ 1 个完成 marker **且** 未命中任何验证 marker → 输出验证提醒。

提醒内容:
```
[my_reverse_skill] 完成度自评提醒:
你说"完成"了,但 transcript 里没看到验证动作。按 08-完成度自评.md 走 6 维:
  1) 代码层: 端到端跑过? 2) 文档层: 自读过? 3) 集成层: 内链查过?
  4) 回归层: 依赖层都跑了? 5) 诚实层: 列了"没验证什么"?
  6) 收尾层: 清理账本和加密算法图补齐?
全打勾 = 10/10; 跳一维 = -3 分。
```

## 与其他规约的关系

| 规约 | 作用域 |
|---|---|
| `04-新增SKILL评分回测准入.md` | 新 Skill 准入门槛 |
| `07-一致性验证规约.md` | 数据层验证(我方产出 vs 真实网页) |
| `11-AI事实证据规约.md` | 防虚假结论,要求关键结论有证据和事实等级 |
| `12-反泛化与任务收敛规约.md` | 防以点概面和发散,要求范围账本 |
| `13-并发指纹与会话隔离规约.md` | 防并发/session/cache 混用和未验证稳定性声明 |
| `14-知识图谱行程与关联规约.md` | 管节点/边/关联/下游依赖更新 |
| `15-AI变更风险与回归校验规约.md` | 管改 A 伤 B、写死请求头/指纹、数据偏移、反扒误处理的回归 |
| `17-交付收尾清理与加密算法图谱规约.md` | 管最终清理、临时/历史/废弃物删除和整体加密算法图 |
| **`08-完成度自评.md`(本文件)** | 工作流验证(Claude 自评是否真完成) |

三者互补: 04 管准入, 07 管数据正确性, 08 管交付质量。
